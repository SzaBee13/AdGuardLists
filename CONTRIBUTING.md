# Contributing Guide

Hey there 👋

First off, thanks for even thinking about contributing. That already makes you cool.

This guide is here to help you get started and make the process as smooth as possible. Whether you're fixing a typo or adding a new feature, your contribution is valuable new list.

## 🚀 Getting Started

1. Fork the repository
2. Clone your fork

   ```bash
   git clone https://github.com/<your-username>/<repo-name>.git
   ```

3. Create a new branch

   ```bash
   git checkout -b feature/your-feature-name
   ```

## 🧠 What You Can Contribute

* Adding new Block/UnBlock lists
* Website improvements
* Fixing bugs
* Improving documentation
* Grammar
* Anything else you think would make this project better

If you're unsure, just open an issue and ask.

## 📏 Coding Guidelines

Keep it clean. Keep it readable.

* Use consistent formatting
* Follow existing code style
* Comment when necessary

## 📬 Pull Requests

When you're ready:

1. Push your branch
2. Open a Pull Request
3. Describe what you changed and why

### PR Tips

* Keep PRs focused (one feature/fix per PR)
* Link related issues if any
* Be clear, not fancy

## 💬 Communication

Be respectful. Be constructive.

We’re all here to learn and build cool stuff.
